## 一、项目定位

**DUS（Distributed AI terminal Unified Scheduling System）** 定位：
- 分布式 AI 终端统一调度系统
- 设备端运行 Bridge，通过轮询从云端获取任务
- 云端提供任务管理、Agent 配置、自动化执行
- 轻量级实现，无需多租户/用户系统

## 四、认证方案

### 4.1 API Key 认证

使用 `X-API-Key` Header 进行认证：

```
X-API-Key: <api_key>
```

### 4.2 API Key 类型

| 类型 | 说明 | 用途 |
|------|------|------|
| 全局 API Key | 云端配置文件中定义 | 开发/测试用 |
| Machine API Key | 设备专属 Key | Bridge 设备认证 |

### 4.3 认证中间件逻辑

1. 检查 `X-API-Key` Header
2. 查找对应的 Machine 或全局 Key
3. 无效或缺失 → 401 Unauthorized

#### T3-2：Skill 系统

**数据模型**：
```python
class Skill(Base):
    __tablename__ = "skills"

    id: Mapped[uuid.UUID] = mapped_column(UUID(), primary_key=True)
    name: Mapped[str] = mapped_column(String(100), unique=True)
    description: Mapped[str | None] = mapped_column(Text, nullable=True)
    content: Mapped[str] = mapped_column(Text)  # SKILL.md 格式
    config: Mapped[dict] = mapped_column(JSON, default=dict)
    created_by: Mapped[uuid.UUID] = mapped_column(UUID(), ForeignKey("users.id"))
    created_at: Mapped[datetime] = mapped_column(default=utcnow)
```

### 6.3 第四阶段任务

#### T4-1：Session Resumption

**概念**：任务执行中断后，可以从断点恢复继续执行。

**Bridge 增强**：
- 定期保存执行状态到 `work_dir/.dus_checkpoint.json`
- 中断后重启时检测 checkpoint，继续执行

#### T4-2：文件活动监控

**功能**：
- Bridge 监控 `work_dir` 下的文件变化
- 变化时更新 Issue 的 `last_activity_at`
- 长时间无活动触发提醒

## 八、WebSocket 事件协议

### 8.1 事件类型

```typescript
type WSEventType =
  // Issue 事件
  | "issue:created" | "issue:updated" | "issue:deleted"
  // 评论事件
  | "comment:created" | "comment:updated" | "comment:deleted"
  // Agent 事件
  | "agent:status" | "agent:created" | "agent:archived"
  // 任务事件
  | "task:queued" | "task:dispatched" | "task:progress" | "task:completed" | "task:failed"
  // 收件箱事件
  | "inbox:new" | "inbox:read"
  // 自动巡逻事件
  | "autopilot:run_start" | "autopilot:run_done"
  // 设备事件
  | "machine:heartbeat" | "machine:status";
```

### 8.2 消息格式

```json
{
  "type": "issue:updated",
  "data": {
    "id": "uuid",
    "status": "in_progress",
    "updated_at": "2026-05-11T00:00:00Z"
  }
}
```
